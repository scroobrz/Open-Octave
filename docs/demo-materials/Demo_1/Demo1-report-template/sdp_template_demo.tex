\section{Quantitative analysis and testing}

{\color{blue}This section is more relevant for the final report. However still include a brief section in the Demo 1 Report and list any preliminary test results or future planned testing. Text below is for the final report.}

This section provides a clear summary of the tests designed and run to prove the reliability and usability of the system as a whole, with details allowed in the appendix.  

This section should first outline any testing methods you used (e.g. repeated runs of subsystems, data-logging, naive user testing). 

It should then present relevant quantitative results. If you are using graphs, please make sure they are properly labelled and logically illustrate the point you want to make (e.g. to compare two algorithms).

For each test, the following points should be answered:

\begin{itemize}
    \item Relevance: How is the test relevant to the system (what does the outcome demonstrate)? How does the outcome of the test allows to validate a milestone / affects the project plan ?
    \item Interpretation: What do the actual results say about the system and why (eg. does a 72\% success rate on a classification algorithm mean that the system is robust or on the contrary that it is not)?
    \item Robustness: is the test exhaustive / are all edge cases considered?
\end{itemize}

In your final report all the updated relevant tests that were performed during the project should be presented and accordingly commented. This includes the Demo 2 User Study. 
This section should convince the reader of the validity of the decisions made and the reliability and/or usability of the system. At an absolute minimum (for the final report), this section should provide a table summarising the  results of tests you have run (for instance, table~\ref{tab:sample-table}, which shows success rates for repeated runs of the whole system). If you present algorithms, you can use the \verb+algorithm+ and \verb+algorithmic+ environments to format pseudocode (for instance, Algorithm~\ref{alg:example}). These require the corresponding style files, \verb+algorithm.sty+ and \verb+algorithmic.sty+ which are supplied with this package. 

\begin{algorithm}[ht]
\begin{algorithmic}
   \STATE {\bfseries Input:} data $x_i$, size $m$
   \REPEAT
   \STATE Initialize $noChange = true$.
   \FOR{$i=1$ {\bfseries to} $m-1$}
   \IF{$x_i > x_{i+1}$} 
   \STATE Swap $x_i$ and $x_{i+1}$
   \STATE $noChange = false$
   \ENDIF
   \ENDFOR
   \UNTIL{$noChange$ is $true$}
\end{algorithmic}
  \caption{Bubble Sort}
  \label{alg:example}
\end{algorithm}

\section{Latex tips}
Here are some common latex errors and how to fix them: 
\begin{description}
    \item[Quotes] To use quotation marks in latex ``like this'', you should use \verb+``+ (two backquote marks) and \verb+''+(two apostrophe marks). 
    \item[References:] To reference tables in your text you should type \verb+table~\ref{tab:sample-table}+ and also label the table with \verb+\label{tab:sample-table}+ in the caption. 
    \item[Citations: ] Citations should always go \emph{inside} the sentence they are relevant to, i.e., before the full stop. 
    \item[Description lists:] This is a description list. Sometimes a description list is more appropriate than a bullet list, and you create it using \verb+\begin{description}+ and \verb+\end{description}+ instead of \verb+\begin{itemize}+.
\end{description}
